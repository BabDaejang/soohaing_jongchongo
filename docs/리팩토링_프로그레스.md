> 실행 지시서는 `docs/리팩토링.md`(v2 — [공통 프롬프트] + [배치 1~4 프롬프트] 구조). 이 문서는 **실제 진행의 누적 기록**이며, 배치 간 유일한 인수인계 통로다.
>
> **진행 방식**: 배치마다 **새 대화**에서 실행한다. 새 대화에는 리팩토링.md의 [공통 프롬프트] + 해당 [배치 N 프롬프트]를 붙여넣거나, 파일 접근이 되는 환경이면 "docs/리팩토링.md의 공통 프롬프트를 따르고 배치 N을 실행해"라고 지시한다. 각 배치는 검증(test/typecheck/lint/build) 통과 → 이 문서 기록 → DECISIONS 기록 → **커밋·main 푸시**까지 마치고 대화를 끝낸다.
>
> **기록 규칙**: 배치를 끝낼 때마다 ① 아래 체크리스트 갱신 ② "진행 기록" 절에 `날짜 / 배치 / 결과 / 검증 출력 요약 / 특이사항(계획과 달라진 점·발견한 문제·다음 배치 인계)`를 **추가**한다(누적만 — 기존 기록 수정·삭제 금지). 계획과 다르게 구현했으면 반드시 사유를 남기고, 설계 자체가 바뀌면 리팩토링.md도 함께 고친 뒤 그 사실을 기록한다.
>
> **배치 시작 전 확인**: 직전 배치가 체크리스트에서 완료 상태인지, 워킹 트리가 깨끗한지(`git status`) 확인. 어긋나면 작업을 멈추고 사용자에게 보고.
