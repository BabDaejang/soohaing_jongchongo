## 진행 기록

### 2026-07-12 / 준비(원인 규명·문서) / 완료

- **PDF "no pages" 원인 실증**: `parsePdfText`(unpdf/pdf.js `getDocumentProxy`)가 입력 Uint8Array의 ArrayBuffer를 detach시킴을 탐침으로 확정 — 호출 전 `bytes.length=328` → 호출 후 **0**, base64 길이 440 → **0**. 스캔 PDF OCR 경로(`ingestDocuments`)가 파싱 **후** `Buffer.from(bytes).toString("base64")`를 하므로 빈 PDF가 전송돼 Google `400 "The document has no pages"` — 모델 무관 재현. `parsePdfText(bytes.slice())` 사본 전달 시 원본 보존 확인. OpenAI PDF 거부는 `toOpenAIContent`의 하드코딩 throw(실제 API는 file 파트로 PDF 지원)로 확인.
- **사용자 선택(2026-07-12) 3건**: ① PDF 수정 = **라이브러리 경계 방어**(parsePdfText 내부 사본 + OCR 빈 데이터 가드 + 회귀 테스트) ② 페이즈 실행 UI를 **대시보드로 통합하고 /ingest·/evaluate 라우트 제거**(리다이렉트, 세부 화면은 유지) ③ 기존 /results 결과 표는 **작업결과표가 흡수·대체**(SPEC 8절 개정 포함).
- 실행 지시서 `docs/리팩토링_2.md`(배치 8개, 공정 순: 버그·기반 → 점수 체계 → 작업결과표 코어 → 상호작용·구화면 정리 → 대시보드·페이즈 0 → 페이즈 1 → 페이즈 2·3 → SPEC·최종)와 이 문서를 생성. 배치 간 고정 계약(앵커 id·워크시트 열 키·`worksheet:refresh` 이벤트·표시 점수 상수·model_routing `default`/`rubric` 키·다운로드 파일명 규칙)을 공통 프롬프트에 명문화 — 하위 모델이 배치를 이어받아도 이름이 어긋나지 않게 함.
