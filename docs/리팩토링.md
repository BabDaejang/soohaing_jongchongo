# 리팩토링 실행 지시서 — 채점 전멸 수정 + 실행 관찰가능성 (v2, 2026-07-11)

> **사용법** — 이 리팩토링은 **배치 4개**로 나뉘며, **배치마다 새 대화**에서 실행한다(맥락 오염·할루시네이션 방지).
> 새 대화에 **[공통 프롬프트] 절 전체 + 해당 [배치 N 프롬프트] 절 전체**를 붙여넣거나, 파일을 읽을 수 있는
> 환경(Claude Code 등)이면 *"docs/리팩토링.md의 공통 프롬프트를 따르고 배치 N을 실행해"* 라고만 지시한다.
> 한 대화 = 한 배치. 배치가 끝나면(커밋·푸시까지) 대화를 종료하고, 다음 배치는 새 대화에서 시작한다.
>
> 배치 구성(작업량 기준): **배치 1** = 1~3단계(원인 제거, 소규모) · **배치 2** = 4~5단계(터미널+채점, 최대 규모)
> · **배치 3** = 6~7단계(매칭+배지, 배치 2를 참조 구현으로 반복) · **배치 4** = 8~9단계(SPEC·최종 검증, 소규모).
> 진행 상태는 `docs/리팩토링_프로그레스.md`가 단일 기준이다.

## [공통 프롬프트] — 모든 배치 대화의 서두에 포함

너는 수행평가 수합·평가·생기부 생성 웹앱(**soohaing_jongchongo** — Next.js App Router + TypeScript strict + Tailwind + Supabase, 배포 Vercel) 저장소에서 `docs/리팩토링.md`의 **배치 하나만** 실행하는 개발자다.

### 시작 절차 (반드시 순서대로)

1. CLAUDE.md 세션 프로토콜대로 `docs/SPEC.md` → `docs/DATA_MODEL.md` → `docs/PROGRESS.md` → `docs/DECISIONS.md`를 읽는다.
2. `docs/리팩토링_프로그레스.md`를 읽고 체크리스트에서 **앞 배치가 완료됐는지 확인**한다. 미완이거나 기록과 실제 코드가 어긋나면 작업을 멈추고 사용자에게 보고한다.
3. `docs/리팩토링.md`에서 지시받은 배치 절만 정독한다. **다른 배치의 작업은 하지 않는다.**

### 배경 (요약 — 상세 근거는 배치 1의 탐침 표)

- 채점 실행이 158건 전멸(2회)했으나 화면에 사유가 전혀 없었다. 원인 확정: ① OpenAI 어댑터가 gpt-5·o 계열이 거부하는 legacy 파라미터(`max_tokens`·`temperature`)를 전송 ② generate/verify에 저장된 `gpt-5.5-pro`는 chat 모델이 아님(404) ③ 에러 메시지가 `catch {}`에서 버려짐 ④ 전건을 한 서버 액션에서 돌리는 구조라 진행 표시·일시정지 불가 + 성공 시 타임아웃 확정.
- 해법 아키텍처 — **클라이언트 구동 1건 단위 실행**: `prepare`(대상 목록·라벨) → `stepOne`(1건 처리, 로그라인 반환) × N → `finalize`(재계산·감사·revalidate). 일시정지 = 다음 호출 안 보냄, 재개 = 이어서, 긴급 중단 = 루프 종료 + finalize(처리분 반영). 재실행 시 증분으로 이어짐(채점 = content_hash, 매칭 = unmatched만). **연속 실패 3건 자동 중단**(서킷 브레이커). 서버 상주 잡·폴링·잡 테이블 없음. **DB 스키마 변경 없음.**
- 사용자 3대 요구: ① 선택 가능 모델 = 실제 호출 가능 모델 ② 선택된 모델을 화면에 표시 ③ 실행 터미널(로그·크기조절·일시정지/재개/긴급중단).

### 불변 규칙 (위반 금지)

- CLAUDE.md의 INV-1~6. 특히 `stepOne`류 액션의 클라이언트 입력은 **id뿐**, 내용·루브릭·명단은 전부 서버가 DB에서 조립한다(INV-2).

### 1단계 — OpenAI 어댑터 현대화

**파일**: `lib/llm/adapters/openai.ts` (현재 43줄짜리 fetch 어댑터)

순수 헬퍼 둘을 export로 추가하고 어댑터 본문이 이를 쓰게 한다:

```ts
// gpt-5·o 계열(추론 모델)은 legacy 파라미터를 거부한다 (2026-07-11 탐침 확정):
//   max_tokens → 400 "Use 'max_completion_tokens' instead"
//   temperature(기본값 외) → 400 "Only the default (1) value is supported"
// 커스텀 openai 호환 엔드포인트(구형)와 gpt-4o 계열은 legacy 파라미터를 유지한다.
export function isOpenAIReasoningFamily(model: string): boolean {
  return /^(gpt-5|o\d)/.test(model);
}

export function buildOpenAIChatBody(input: {
  model: string;
  messages: unknown[];       // toOpenAIContent 매핑이 끝난 배열
  maxTokens: number;
  temperature?: number;
}): Record<string, unknown> {
  const { model, messages, maxTokens, temperature } = input;
  if (isOpenAIReasoningFamily(model)) {
    return { model, messages, max_completion_tokens: maxTokens }; // temperature 생략(탐침 D)
  }
  const body: Record<string, unknown> = { model, messages, max_tokens: maxTokens };
  if (temperature !== undefined) body.temperature = temperature;
  return body;
}
```

**빈 응답 방어** — 추론 토큰이 한도를 소진하면 content가 비는데 지금은 조용히 전 기준 0점 저장됨. 응답 파싱 후:

```ts
const choice = data.choices?.[0];
const text = choice?.message?.content ?? "";
if (!text.trim()) {

  );
}
```

### 4단계 — 공용 실행 터미널

**신규** `lib/hooks/use-sequential-run.ts` (`"use client"`):

```ts
export type TermLevel = "info" | "ok" | "error" | "system";
export type TermLine = { ts: string; level: TermLevel; text: string }; // ts = "14:03:05" (ko-KR, hour12:false)
export type RunState = "idle" | "running" | "paused" | "stopping" | "done" | "aborted";
export type SequentialTarget = { id: string; label: string };
export type PrepareResult = {
  targets: SequentialTarget[];
  prelude?: { level: TermLevel; text: string }[]; // 대상 요약·결정적 처리 결과 등 선행 로그
};
export type StepResult = { ok: boolean; message: string };

export function useSequentialRun(args: {
  prepare: () => Promise<PrepareResult>;
  stepOne: (t: SequentialTarget) => Promise<StepResult>;
  finalize: (r: { succeeded: number; failed: number; aborted: boolean }) => Promise<string | null>;
  maxConsecutiveFailures?: number; // 기본 3
}): {
  lines: TermLine[];
  runState: RunState;
  progress: { done: number; total: number } | null;
  start: () => void; // lines 초기화 후 루프 시작. runningRef로 이중 시작 방지
  pause: () => void; // pauseRef=true — 진행 중 1건은 완료되고 다음 스텝 직전에 멈춤
  resume: () => void;
  stop: () => void;  // stopRef=true — 다음 스텝 직전 루프 종료 → finalize(aborted:true)
};
```

구현 요점: pause/stop은 **ref 플래그**(루프에서 직접 읽음, pause 중 250ms poll sleep). 스텝 로그는 `` `${t.label} — ${r.message}` ``(ok→"ok", 실패→"error"). 연속 실패 `maxConsecutiveFailures`건이면 system 라인("연속 실패 3건 — 자동 중단") 후 aborted 종료. 일시정지/재개/중단도 system 라인 기록. prepare/finalize의 throw는 error 라인 + aborted. finalize 반환 문자열은 system 라인 출력.

**신규** `components/projects/run-terminal.tsx` (`"use client"`, 표시 전용 — **채점 전용 명칭·로직 금지**, 배치 3에서 매칭이 재사용):

- props: `{ lines, runState, progress, onPause, onResume, onStop }`.
- 로그 창: `font-mono text-xs bg-zinc-950 text-zinc-200 rounded-md p-3 h-56 min-h-32 max-w-full resize overflow-auto` — **Tailwind `resize`로 크기 조절**(요구 ③). 각 줄 `[HH:MM:SS]` 프리픽스, level 색: ok=`text-emerald-400` / error=`text-red-400` / system=`text-amber-300` / info=`text-zinc-300`.
- 자동 스크롤: lines 변경 시 `el.scrollTop = el.scrollHeight`.
- 헤더: 진행 카운터 `{done}/{total}` + running 중 [일시 정지]·[긴급 중단], paused 중 [다시 시작]·[긴급 중단].

### 5단계 — 채점 전환

```ts
export type EvalTarget = { id: string; label: string }; // label = "학번 이름"(+ " · 파일명")

export async function prepareEvaluation(projectId: string): Promise<{ targets: EvalTarget[]; skipped: number }>;
// requireProjectOwner. 루브릭 기준 0개면 throw. 대상 조건 기존과 동일(include_in_eval + student_id + auto_matched/confirmed).
// label용 학생 정보는 students 1회 조회로 조인. 증분: admin으로 evaluations(is_current) content_hash 비교(기존 로직 이동).
```
